# 项目结构说明

本文档说明当前项目在工作流合并后的结构。

## 主要层级

### `ui_app.py`
面向操作者的应用界面。

职责：
- 接收用户输入
- 展示三步工作流
- 显示任务行和素材状态
- 触发生成视频动作

非职责：
- 不定义剪辑规则
- 不负责 TTS 底层实现
- 不承载过深的工作流逻辑

### `src/workflow.py`
核心工作流层。

职责：
- 校验结构化剧本
- 生成拍摄任务行
- 推导 UI 所需任务 metadata
- 处理素材命名辅助逻辑
- 汇总素材覆盖情况
- 准备内部 production YAML
- 补写最终渲染设置

这个文件是当前产品逻辑的主要承载层。

### `src/director_engine/`
导演规则层。

职责：
- 应用节奏规则
- 控制重复
- 处理转场
- 处理结尾逻辑

这一层负责修改时间线风格，但不负责定义产品工作流本身。

### `src/main.py`
CLI 入口。

职责：
- 提供 compile / guide / run 命令
- 将工作流逻辑与渲染逻辑连接起来

### TTS / 字幕 / 配音相关文件
- `src/tts_provider.py`
- `src/voiceover_a2.py`
- `src/subtitle_builder.py`

这些文件属于引擎级支撑模块。

## 输入目录

### `creative_scripts/`
存放结构化剧本输入和示例。

### `data/tts_profiles/`
存放本地 ElevenLabs 配置。

## 输出原则

输出目录保持轻量。

一次运行通常只保留必要内容，例如：
- creative script 快照
- 任务行
- 渲染日志
- 最终视频

仓库不再把大量调试中间产物当作正式输出的一部分。
