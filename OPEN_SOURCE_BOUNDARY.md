# Open Source Boundary

This document defines the practical boundary for what this repository plans to
open and what should remain controlled or handled with caution.

## Planned to be open

- General-purpose code in this repository.
- Workflow definitions and process wiring.
- Contract/schema-related structures used for governed behavior.
- Foundational UI layers and interaction patterns.
- Example configuration files intended for learning or local setup.

## Not automatically open (or requires caution)

- Client materials, customer assets, and private operational data.
- Unauthorized third-party resources or content with unclear rights.
- Secrets, tokens, keys, credentials, and local environment artifacts.
- Brand assets, including reserved names, logos, and visual identity items.
- Separately maintained rule packs, templates, datasets, or brand-specific
  assets that are not part of the default public repository scope.

When preparing a public release, each artifact should be reviewed against these
boundaries before publication.
