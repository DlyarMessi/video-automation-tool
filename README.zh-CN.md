# 剧本优先的视频自动化工具

这是一个面向工业宣传短视频生产的三步工作流系统。

1. 提供结构化剧本  
2. 生成拍摄任务并补齐素材  
3. 合成最终视频

本项目的目标，是把重复性的视频生产流程变成更清晰、更稳定、可持续优化的内部工作流。

## 核心理念

系统采用“剧本优先”的设计方式。

结构化剧本是唯一核心输入。  
基于这一份输入，系统可以继续完成：

- 生成拍摄任务
- 整理素材需求
- 准备时间线逻辑
- 应用剪辑规则
- 输出最终视频

## 当前工作流

### 第一步：剧本 → 任务
操作者提供 YAML 格式的创意剧本。

系统会解析剧本，并生成：

- 拍摄任务行
- 场景 / 类别 / 镜头拆解
- 缺失素材需求
- 可选的可打印任务输出

### 第二步：素材面板
操作者查看所需素材，并补齐缺失镜头。

这一步支持：

- 检查现有工厂素材
- 识别缺失项
- 上传新素材
- 将素材从待整理区移入工厂素材池
- 按统一命名规则整理素材

### 第三步：生成视频
系统准备内部 production timeline，应用渲染设置，并生成最终视频。

## 项目结构

- `ui_app.py` — 面向操作者的 UI
- `src/workflow.py` — 三步工作流核心逻辑
- `src/main.py` — CLI 入口
- `src/director_engine/` — 导演规则层
- `src/tts_provider.py` — ElevenLabs TTS 提供层
- `src/voiceover_a2.py` — 配音事件处理
- `src/subtitle_builder.py` — 字幕构建
- `creative_scripts/` — 剧本输入与示例
- `data/tts_profiles/` — 本地 TTS 配置

更详细说明见 `PROJECT_STRUCTURE.zh-CN.md`。

## 设计原则

- 剧本优先
- 稳定优先于炫技
- 可复用优先于一次性产物
- 工作流清晰优先于实验性复杂结构
- 每次运行输出尽量轻量

## 说明

- 当前只支持 ElevenLabs 作为 TTS 提供方
- 素材盘可以位于外置存储设备
- 当素材存储不可用时，UI 会尽量安全降级
- 每次运行的输出目录保持轻量

## 下一阶段方向

后续重点不再是仓库清理，而是继续优化主工作流：

- 更稳定的剧本 schema
- 更清晰的素材槽位逻辑
- 给导演引擎提供更好的 metadata
- 完整的中英双语文档
- 后续支持中文 UI
