# Contributing

Thanks for your interest in improving this project.

## Current stage expectations

- Issues and pull requests are welcome.
- Keep each pull request focused on a single objective whenever possible.
- Do not bypass governed schema/contract layers when adding or changing logic.
- Avoid making the UI heavier just to maximize feature completeness.
- Provider expansion is not the first priority at the current stage.
- Only submit content you have the legal right to contribute.

## Practical guidance

- Keep changes small, deterministic, and reviewable.
- Prefer additive changes over broad refactors.
- Open an issue first before proposing broad structural changes.
- Include tests when behavior, contracts, or validation logic changes.
