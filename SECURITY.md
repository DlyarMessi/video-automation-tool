# Security Policy

## Reporting

Please do not publish vulnerability details in public issues, pull requests, or
discussion threads.

For security concerns, please contact the maintainer privately at:
[YOUR-SECURITY-EMAIL]

## Safe contribution reminders

- Do not commit secrets, tokens, credentials, or customer data.
- Do not commit local machine configuration or private provider workspace data.
- Check `.gitignore` before committing new files.
- Re-check local workspace/provider settings before pushing.
